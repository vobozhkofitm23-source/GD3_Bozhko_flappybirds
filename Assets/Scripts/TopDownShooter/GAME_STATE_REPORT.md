## Додані механіки взаємодії

1. **HP (здоров'я)** — гравець має 5 одиниць здоров'я. Відображається через `Slider` та `Image` (fill) у UI. Вороги завдають урон при зіткненні.
2. **Рахунок (Score)** — за кожного вбитого ворога нараховується очки. Виводиться через `TextMeshProUGUI`.
3. **Умова програшу** — коли HP = 0, стан гри змінюється на `GameOver`, зупиняється спавн ворогів, з'являється панель програшу (`Image` + `TMP`).
4. **Перезапуск сцени** — клавіша `R` або кнопка `Restart` викликають `SceneManager.LoadScene`.
5. **Dash (уклонення)** — `Space` дає короткий ривок з неуразливістю; `OverlapCircle` відштовхує ворогів поруч.
6. **Health Pickup** — аптечки на карті відновлюють HP через `OnTriggerEnter2D`.

## Система стану гри

```csharp
public enum GameState { Playing, GameOver }
```

- **Playing** — гравець рухається, стріляє, вороги спавняться.
- **GameOver** — ввід зброї блокується, спавн зупинено, показано UI програшу.

Перехід: `Player.OnDeath` → `GameManager.HandlePlayerDeath()` → `GameState.GameOver`.

## 1. Які типи колізій використані?

| Тип | Де | Навіщо |
|-----|-----|--------|
| **Distance check** | `Enemy` ↔ `Player` | Урон при наближенні ворога до гравця |
| **Trigger2D** (`OnTriggerEnter2D`) | `Projectile` ↔ `Enemy` | Куля влучає у ворога без фізичної відповіді |
| **Trigger2D** (`OnTriggerEnter2D`) | `HealthPickup` ↔ `Player` | Підбір аптечки |

## 2. Де застосовано Raycast або Overlap?

| Метод | Файл | Призначення |
|-------|------|-------------|
| `Physics2D.Raycast` | `Laser.cs` | Лазер визначає першу ціль уздовж променя |
| `Vector2.Distance` | `Enemy.cs` | Ворог бачить гравця та атакує в радіусі |
| `Physics2D.OverlapCircleAll` | `Player.cs` | Dash відштовхує ворогів у радіусі |

## 3. Чому обрано саме такий метод перевірки?

- **Raycast (лазер)** — потрібна точна лінія у напрямку пострілу; raycast дає першу перешкоду на траєкторії.
- **Distance (агро/атака ворога)** — надійніше за `OverlapCircle`, який повертав лише один випадковий колайдер.
- **OverlapCircleAll (dash)** — ривок впливає на всіх ворогів у зоні одночасно, а не на одну ціль.
- **Trigger (снаряд/аптечка)** — об'єкт проходить крізь ціль і одразу зникає; trigger не блокує рух.
- **Distance (урон)** — kinematic-тіла не викликають `OnCollisionEnter2D`, тому урон перевіряється по відстані.

## 4. Як механіки впливають на core loop

**Core loop:** рухатися → стріляти у ворогів → набирати очки → виживати.

- **HP + вороги** — помилки коштують здоров'я; гравець не може лише стояти на місці.
- **Score** — мотивує активно вбивати ворогів, а не лише уникати.
- **Dash + Health Pickup** — дають способи відновити/зберегти HP, додають тактичний вибір.
- **GameOver + Restart** — замикає цикл: програш → повторити спробу покращити результат.
- **Detection range** — вороги не гоняться з іншого кінця карти; напруга локалізована біля гравця.
